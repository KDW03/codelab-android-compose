UI나 화면 테스트는 Compose 코드의 올바른 동작을 확인하는 데 사용되므로 개발 프로세스 초기에 오류를 포착하여 앱 품질을 개선

Compose는 요소를 찾아 속성을 확인하고 사용자 작업을 실행하는 일련의 테스트 API를 제공



격리 테스트, 디버깅 테스트, 시맨틱 트리, 동기화

Compose를 사용하면 개별 컴포저블을 테스트하고 다른 컴포저블과의 상호작용을 제어할 수 있어서 테스트 작업이 더 간단

createComposeRule()은 ComposeTestRule을 생성하는 함수
이 규칙을 사용하면 테스트 중인 Compose 콘텐츠를 설정하고 이와 상호작용

composeTestRule은 테스트 규칙을 나타내며
이 규칙을 사용하여 UI 요소를 찾고(.finder),
해당 요소의 속성을 확인(.assertion)하고,
작업을 실행(.action)할 수 있다.

시맨틱 트리
Compose UI 테스트에서 시멘틱 트리는 앱의 UI 구조를 나타내는 트리
각 노드는 UI 요소와 해당 요소의 속성을 나타냄
시멘틱 트리를 사용하면 테스트 코드에서 UI 요소를 쉽게 찾고, 해당 요소의 속성을 확인하고, 작업을 실행
